# Wellman Fitness AI

Wellman is a high-performance, AI-powered fitness application designed for modern health tracking. It integrates Google Gemini's advanced multimodal capabilities and machine learning to provide intelligent fitness guidance, vision-based body metrics, nutritional analysis, and posture assessment.

## 🚀 Key Features

- **Smart Dashboard**: Real-time weight tracking, consistency metrics, and active fitness plans with historical data visualization.
- **AI Fitness Plan Designer**: Personalized workout plans generated by Gemini based on user goals and fitness level.
- **AI Nutritionist**: Vision-based meal analysis providing caloric and macro-nutrient estimates.
- **BMI/Body Estimator**: Advanced volumetric body metric analysis using computer vision.
- **Posture Checker**: AI-powered posture analysis with alignment assessment and corrective recommendations.
- **Admin Dashboard**: Comprehensive platform management, user analytics, and system monitoring.

## ⚡ Quick Start

### Prerequisites
- **Python 3.10+**
- **Node.js 18+** with npm
- **Google Gemini API Key** (free tier available)

### Automatic Setup (Recommended)

1. **Clone or extract the project**:
   ```bash
   cd wellman-fitness-version-1.3.6
   ```

2. **Create environment files from examples**:
   ```bash
   copy .env.example .env
   copy backend\.env.example backend\.env
   ```

3. **Configure environment variables**:
   Edit `.env` in root and `backend/.env` with your values (see Environment Variables section below).

4. **Start all services**:
   ```bash
   # Terminal 1 - Backend
   cd backend
   python -m venv venv
   venv\Scripts\activate
   pip install -r requirements.txt
   python manage.py migrate
   python manage.py createsuperuser

   # Terminal 2 - Frontend
   npm install
   npm run dev
   ```

### Manual Setup

If the automated scripts fail, follow these steps:

**Terminal 1 (Backend)**:
```bash
cd backend
python -m venv venv
venv\Scripts\activate
pip install -r requirements.txt
python manage.py migrate
python manage.py createsuperuser  # Create admin account
python manage.py runserver
```

**Terminal 2 (Frontend)**:
```bash
npm install
npm run dev
```

## 🐍 Django Backend Setup

### Database Configuration

The system uses SQLite by default (suitable for development) and supports PostgreSQL for production.

### Models & Database Schema

The backend includes the following main models:

```python
# User Management
- User (extends Django's AbstractUser)
  - display_name, bio, avatar_seed
  - height_cm, weight tracking
  - fitness_profile (JSON field for custom data)

# Fitness Tracking
- WeightLog: Historical weight records
- FitnessPlan: User-specific workout plans
- ActivityLog: Daily activity records
- PostureLog: Posture assessment data with scores

# Nutrition Tracking
- MealLog: Meal records with macro breakdown
  - calories, protein, carbs, fat
```

### Initial Setup

```bash
# Navigate to backend directory
cd backend

# Create virtual environment
python -m venv venv

# Activate virtual environment (Windows)
venv\Scripts\activate

# Install dependencies
pip install -r requirements.txt

# Create and apply migrations
python manage.py makemigrations api
python manage.py migrate

# Create admin/superuser
python manage.py createsuperuser
# Use: username=admin_jafitness, password=admin123

# (Optional) Seed database with test data
python seed.py
```

Copy `.env.example` to `.env` in the `backend/` directory:

## 🚀 Deployment

### Development
Use `npm run dev` and `python manage.py runserver` with automatic hot-reloading.

### Production (Vercel + Railway)

#### Frontend (Vercel)
1. **Build and deploy**:
   ```bash
   npm run build
   vercel deploy --prod
   ```

2. **Environment Variables** (in Vercel dashboard):
   - `VITE_API_KEY` - Google Gemini API key
   - `VITE_API_BASE_URL` - Railway backend URL (e.g., `https://your-app.up.railway.app`)

#### Backend (Railway)
1. **Deploy**:
   - Connect GitHub repository to Railway project
   - Railway automatically deploys on git push to main branch

2. **Environment Variables** (in Railway dashboard):
   - `GEMINI_API_KEY` - Google Gemini API key
   - `SECRET_KEY` - Django secret key
   - `DEBUG=False` - Production mode
   - `ALLOWED_HOSTS` - Your Vercel domain
   - `CORS_ALLOW_ALL_ORIGINS=True` - Allow Vercel frontend

#### CORS Configuration

The backend uses `django-cors-headers` to allow cross-origin requests from the Vercel frontend.

**Settings in `backend/settings.py`:**
```python
# Allow all origins (recommended for development)
CORS_ALLOW_ALL_ORIGINS = True

# Or explicitly whitelist Vercel domains
CORS_ALLOWED_ORIGINS = [
    "https://your-app.vercel.app",
]

# Use regex for Vercel preview deployments
CORS_ALLOWED_ORIGIN_REGEXES = [
    r"https://your-app-.*\.vercel\.app",
]

# Allow credentials (cookies)
CORS_ALLOW_CREDENTIALS = True
```

**Railway Configuration (`railway.toml`):**
```toml
[deploy.env]
CORS_ALLOW_ALL_ORIGINS = "True"
CORS_ALLOWED_ORIGINS = "https://your-app.vercel.app"
```

### Manual Production Deploy
1. **Frontend**: Build and serve with static hosting
   ```bash
   npm run build
   # Serve dist/ folder with nginx or Apache
   ```

2. **Backend**: Use gunicorn with reverse proxy
   ```bash
   gunicorn backend.wsgi:application --bind 0.0.0.0:8000
   ```

3. **Database**: Migrate to PostgreSQL for production
4. **Caching**: Configure Redis for sessions and Celery tasks
5. **Environment**: Set `DEBUG=False` and update `ALLOWED_HOSTS`

## 🐛 Troubleshooting

### Common Issues

**Port Already in Use**
```bash
# Find and kill process on port 8000
netstat -ano | findstr :8000
taskkill /PID <PID> /F
```

**Missing Dependencies**
```bash
# Reinstall all dependencies
pip install --upgrade pip
pip install -r requirements.txt
npm install
```

**Database Errors**
```bash
# Reset database and migrations
rm db.sqlite3
rm backend/api/migrations/0*.py
python manage.py makemigrations api
python manage.py migrate
```

**API Key Not Working**
- Verify API key is set in `.env` files
- Check Google Cloud Console for correct API key
- Ensure Gemini API is enabled in your Google project
